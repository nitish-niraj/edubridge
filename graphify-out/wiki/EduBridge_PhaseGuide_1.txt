Project Overview & How to Use This Guide
This document provides granular, AI-agent-ready instructions for building EduBridge � a web platform connecting retired teachers with students seeking personalized tutoring. Every phase is self-contained and can be handed directly to an AI coding agent.

How to Use This Document
� Each phase has a GOAL, DURATION, TECH CONTEXT, and a TASK TABLE.
� The task table gives every individual component with a full description column � this is what you paste to an AI agent.
� Phases must be completed in order. Each phase depends on the previous phase's output.
� All backend is Laravel 10 (PHP 8.2). All frontend uses Vue.js 3 embedded in Laravel Blade templates.
� Database: MySQL 8. Real-time: Pusher + Laravel Echo. Video: Twilio. Payments: Razorpay.

Global Technical Standards (Apply to ALL Phases)
� Project name: edubridge. Laravel project lives at /var/www/edubridge on the server.
� All PHP must be PSR-12 compliant. All Vue components use the Composition API (setup()).
� Every API route must return JSON. Use Laravel API Resources for all responses.
� All DB migrations must use up() and down() methods properly.
� Every controller method must have Form Request validation � no raw $request->input() without validation.
� Soft deletes (SoftDeletes trait) on all major tables: users, bookings, messages.
� All timestamps in UTC in the database. Display conversion happens on the frontend.
� Use .env for all secrets � never hardcode API keys.
� Write a Feature Test for every major API endpoint using PHPUnit.

Database Naming Conventions
� Table names: snake_case plural (teacher_profiles, booking_slots).
� Foreign keys: table_singular_id pattern (teacher_id, student_id, booking_id).
� JSON columns: use MySQL JSON type for arrays like subjects[], languages[].
� Enum columns: use string with check constraints via Laravel enum cast.

PHASE 1: Foundation � Laravel Setup, DB Schema, Authentication & Profiles
Duration: Months 1�2  |  Goal: Get the skeleton of the app running with working login and profile pages

Phase 1 Overview
This is the most critical phase. Every other phase depends on a clean, well-structured foundation. The AI agent must set up the entire Laravel project from scratch, configure the database, build the role-based authentication system, and create working profile pages for both Teachers and Students. By the end of Phase 1, a user should be able to register as either a student or a teacher, verify their email, log in, and fill out their complete profile.

Phase 1 � Environment & Stack

� Laravel Telescope (dev only) for debugging
